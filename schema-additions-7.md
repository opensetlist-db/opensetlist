## Correction 3 — Slug auto-generation for user contributions

slug is stored in DB (Artist, Song, Event, EventSeries, Album).
Admin CSV import specifies slugs explicitly.
User-contributed data must have slugs auto-generated by the server.

### Rule: Users never see or input slug

```
User contribution form:
  曲名: [ハナムスビ        ]
  アーティスト: [スリーズブーケ ▾]
  発売日: [2023-05-10     ]
  [Submit]

Server automatically:
  slug = generateUniqueSlug("ハナムスビ", "song")
       = "hanamusubi"

User never sees or inputs slug.
Admin can view/edit slug in admin UI if needed.
```

### generateSlug() — base function

Add to `src/lib/slug.ts`:

```typescript
/**
 * Converts a string to a URL-safe slug.
 * ASCII only — non-ASCII characters are stripped after processing.
 * For Japanese/Korean names, use generateSlugFromJapanese() instead.
 */
export function generateSlug(input: string): string {
  return input
    .toLowerCase()
    .replace(/[^\w\s-]/g, '')   // remove non-word chars except hyphens
    .replace(/[\s_]+/g, '-')    // spaces/underscores → hyphens
    .replace(/-+/g, '-')        // consecutive hyphens → single hyphen
    .replace(/^-|-$/g, '')      // trim leading/trailing hyphens
    .slice(0, 100)
}

// Examples:
// generateSlug("Dream Believers")    → "dream-believers"
// generateSlug("DOLLCHESTRA")        → "dollchestra"
// generateSlug("Holiday∞Holiday")   → "holidayholiday"
// generateSlug("ハナムスビ")         → ""  (Japanese stripped)
```

### generateUniqueSlug() — with Japanese support and dedup

Add to `src/lib/slug.ts`:

```typescript
import { prisma } from '@/lib/prisma'

type SlugModel = 'artist' | 'song' | 'event' | 'eventSeries' | 'album'

/**
 * Generates a unique slug for a given model.
 * Handles Japanese/Korean via transliteration.
 * Appends incrementing number if slug already exists.
 */
export async function generateUniqueSlug(
  input: string,
  model: SlugModel
): Promise<string> {
  // Step 1: Try direct ASCII slug
  let base = generateSlug(input)

  // Step 2: If empty (Japanese/Korean), transliterate first
  if (!base) {
    base = generateSlug(await transliterateToRomaji(input))
  }

  // Step 3: Final fallback — model name + timestamp
  if (!base) {
    base = `${model}-${Date.now()}`
  }

  // Step 4: Ensure uniqueness
  return await ensureUnique(base, model)
}

async function ensureUnique(base: string, model: SlugModel): Promise<string> {
  let candidate = base
  let count = 1

  while (await slugExists(candidate, model)) {
    candidate = `${base}-${count}`
    count++
  }

  return candidate
}

async function slugExists(slug: string, model: SlugModel): Promise<boolean> {
  switch (model) {
    case 'artist':
      return !!(await prisma.artist.findUnique({ where: { slug } }))
    case 'song':
      return !!(await prisma.song.findUnique({ where: { slug } }))
    case 'event':
      return !!(await prisma.event.findUnique({ where: { slug } }))
    case 'eventSeries':
      return !!(await prisma.eventSeries.findUnique({ where: { slug } }))
    case 'album':
      return !!(await prisma.album.findUnique({ where: { slug } }))
  }
}
```

### transliterateToRomaji() — Japanese to romaji

Install dependency:
```bash
npm install kuroshiro kuroshiro-analyzer-kuromoji
```

Add to `src/lib/slug.ts`:

```typescript
import Kuroshiro from 'kuroshiro'
import KuromojiAnalyzer from 'kuroshiro-analyzer-kuromoji'

let kuroshiro: Kuroshiro | null = null

async function getKuroshiro(): Promise<Kuroshiro> {
  if (!kuroshiro) {
    kuroshiro = new Kuroshiro()
    await kuroshiro.init(new KuromojiAnalyzer())
  }
  return kuroshiro
}

/**
 * Converts Japanese text to romaji for slug generation.
 * "ハナムスビ" → "hanamusubi"
 * "上昇気流にのせて" → "joushoukiryuuninosete"
 */
async function transliterateToRomaji(input: string): Promise<string> {
  try {
    const k = await getKuroshiro()
    return await k.convert(input, { to: 'romaji', mode: 'spaced' })
  } catch {
    return ''  // fallback to timestamp if conversion fails
  }
}
```

### Usage in user contribution flow

```typescript
// When user submits a new Song:
const slug = await generateUniqueSlug(formData.originalTitle, 'song')

await prisma.song.create({
  data: {
    slug,
    originalTitle: formData.originalTitle,
    // ...
  }
})

// When user submits a new Artist:
const slug = await generateUniqueSlug(formData.name, 'artist')

// When user submits a new Event:
// Use shortName or city+date as base for readability
const base = `${formData.city}-day${formData.dayNumber}`
const slug = await generateUniqueSlug(base, 'event')
// → "fukuoka-day1", "kobe-day2"
```

     Existing URLs will redirect automatically.
